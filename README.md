🌐 REST API Client Using Servlet, JSP, and Web Technologies :

This Java web application demonstrates how to consume a public REST API (such as weather data) and display the response in a structured and user-friendly format using Servlets, JSP, and basic HTML, CSS, and JavaScript.

🔧 Technologies Used :

-Java Servlets & JSP (for backend processing)

-HTML/CSS/JavaScript (for frontend UI)

-HTTPURLConnection (for API request handling)

-org.json or other libraries (for JSON parsing)

-Apache Tomcat (for deployment and testing)

📌 Project Objective :

The objective of this project is to create a working example of a REST API consumer using traditional Java EE components. The application sends HTTP GET requests to a public API (like OpenWeatherMap), parses the JSON response, and displays selected data (such as temperature, humidity, and weather conditions) on a styled webpage.

📝 Features :

-Accepts user input (e.g., city name) from a web form.

-Makes an HTTP request to a RESTful API endpoint.

-Parses the JSON response using Java.

-Forwards the data to a JSP page for display.

-Basic validation and error handling included.

🔄 Workflow :

-User enters a city name in the HTML form.

-The Servlet captures the input and sends an HTTP GET request to the API.

-The JSON response is parsed in the Servlet.

-Parsed data is passed to the JSP for rendering.

-The result is displayed with styling using CSS and enhanced interactivity using JavaScript.

✅ Deliverable :

-A fully functional Java web application that:

-Handles HTTP requests using Servlets

-Parses JSON responses from a REST API

-Displays structured output using JSP and frontend technologies
